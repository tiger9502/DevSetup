# OpenWRT Setup Instructions for AX1800

## Step 1: Firmware
- Connect to a freshly reset AX1800 router via Wifi or ethernet. The setup interface should be at ```192.168.8.1```.
- In luci, there is an option to upgrade the firmware with OpenWRT versions. You may find the update images on the manufacturer site [here](https://dl.gl-inet.com/router/ax1800/stable). **This guide assumes a firmware version of 4.6.8, as it is the last version for easytether to work**.
- Once flashed, you will need to connect to 192.168.8.1 via an ethernet connection. Set admin password, Wifi SSID and passwords.
- From now on we will be using luci for the rest of the setup. It should be accessible at ```192.168.8.1/cgi-bin/luci```
- Go to System -> General Settings and perform a time sync (Sync with NTP-Server).
- Assuming you need to connect the router to a Hotspot for internet access:
  * In Network -> Wireless, use the 5G radio's Scan to connect to the hotspot.
  * Disable all wireless connections for both radio except for the Client mode connection. You should get a reading of the signal strength (blue bars).
  * Verify internet access using Network -> Diagnostics.

## Step 2: Installations and Configurations
- SSH into the router with ```ssh -o HostKeyAlgorithms=+ssh-rsa root@192.168.8.1``` and the admin password.
- Install the following useful packages:
  ```
  opkg update &&
  opkg install unzip openssh-sftp-server openssl-util kmod-usb-net-rndis kmod-usb-storage usbutils
  ```
- Download and install easytether driver:
  ```
  wget http://www.mobile-stream.com/beta/openwrt/easytether-usb-tiny_0.8.9-5_openwrt-19.07.3.zip &&
  unzip easytether-usb-tiny_0.8.9-5_openwrt-19.07.3.zip &&
  tar -zxf 19.07.3/bcm53xx/generic/easytether-usb-tiny_0.8.9-5_arm_cortex-a9.ipk &&
  tar -zxf data.tar.gz -C /
  ```
- Add an adbkey as follows:
  ```
  mkdir -p /etc/easytether &&
  openssl genpkey -algorithm rsa -pkeyopt rsa_keygen_bits:2048 -out /etc/easytether/adbkey &&
  chown root:root /etc/easytether/adbkey &&
  chmod 600 /etc/easytether/adbkey
  ```
- Edit /etc/config/network, and make sure the relevant sections contain the correct options:
  * config interface 'wan' and 'wan6' should contain (proto may be'dhcpv6' for wan6):
    ```
    option proto 'dhcp'
    option ifname 'tap-easytether'
    ```
  * create an interface 'easytether':
    ```
    config interface easytether
        option proto 'dhcp'
        option metric '30'
        option ifname 'tap-easytether'
    ```
- Create file ```/etc/config/firewall.user```:
  ```
  iptables -t mangle -I POSTROUTING -o usb0 -j TTL --ttl-set 65
  ```
- Modify ```/etc/config/firewall```:
  * add the following:
    ```
    config include
      option path '/etc/config/firewall.user'
      option fw4_compatible '1'
    ```
  * Add an additional line after ```list network 'wwan'``` in the zone with name 'wan':
    ```
    list network 'easytether'
    ```
- Restart the firewall: ```/etc/init.d/firewall restart```

## Step 3 Watch Scripts and Misc
- Create ```/etc/config/resettether``` (Assuming the connecting device is Pixel 6a. You can find out the device name with the command ```lsusb -v | grep iProduct```):
  ```
  for i in $(seq 1 60);
  do
      if (usbreset "Pixel 6a" || usbreset 001/002)
      then exit 0
      fi
      sleep 5
  done

  reboot -f
  ```
- Create ```/etc/config/easytether```:
  ```
  for i in $(seq 1 20);
  do
      if (ip a s tap-easytether up); then
          :
      else
          sh /etc/config/resettether && easytether-usb
      fi
      sleep 3
  done
  ```
- Create ```/etc/config/watchdog```:
  ```
  . /lib/functions/network.sh
  network_flush_cache
  network_find_wan NET_IF
  network_get_gateway NET_GW "${NET_IF}"

  for i in $(seq 1 5);
  do
      if ping -c 1 -w 3 "${NET_GW}" &> /dev/null
      then exit 0
      fi
  done

  sh /etc/config/resettether
  ```
- Edit the cron jobs via ```crontab -e```:
  ```
  * * * * * sh /etc/config/easytether
  * * * * * sh /etc/config/watchdog
  ```
- (Optional) Block Apple MDM enrollment:
  ```
  echo "0.0.0.0 iprofiles.apple.com" >> /etc/hosts
  echo "0.0.0.0 mdmenrollment.apple.com" >> /etc/hosts
  echo "0.0.0.0 deviceenrollment.apple.com" >> /etc/hosts
  echo "0.0.0.0 gdmf.apple.com" >> /etc/hosts
  ```

## Finish
- Go to Network -> Wireless. Enable all wireless interfaces in Master mode, set SSID and wifi password (recommended security: WPA2-PSK). Note that you should set different SSIDs for 2.4G and 5G networks. Remove the Client wireless interface.
- Go to Network -> Interface. Make sure the LAN IPv4 address is ```192.168.1.1``` and netmask ```255.255.255.0```
